## 问题2：文_是否为空 映射不准确

**文件**：`D:\Tools\Dao\dao\runtime.py`  
**位置**：`_inject_builtins()` 中的 `_cn_aliases` 字典

**现象**：`文_是否为空` 映射到了 `is_str`（检查是否为字符串类型），但 `.ku` 文件里的用法是判断字符串内容是否为空/空白：
```
若 文_是否为空(章) { 续 }
```
意思是"如果章的内容是空的，就跳过"。`is_str` 只能判断类型，不能判断内容是否为空。

**修复方案**：
1. 在 `_inject_builtins()` 的 Python 函数定义区添加：
```python
def _str_is_empty(s):
    return not s or (isinstance(s, str) and s.strip() == "")
```
2. 注册为内置 thought：
```python
"str_is_empty": Thought("str_is_empty", ["s"], Node.lit(None), doc="检查字符串是否为空或空白"),
```
3. 在 `_cn_aliases` 中将 `"文_是否为空"` 改为 `"str_is_empty"`

**状态**：⏳ 待修复

## 问题3：_inject_builtins 中重复创建了已有的 Thought 对象

**文件**：`D:\Tools\Dao\dao\runtime.py`  
**位置**：`_inject_builtins()` 末尾（约第2291行）

**现象**：运算符 builtins（`+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `>`, `<=`, `>=`）和逻辑运算符（`and`, `or`, `not`）通过 `Thought()` 构造函数创建并注册到 `Thought.registry`，但它们的 `.meta["builtin"]` 标记是在后面的循环中才设置的。如果这些 thought 通过 `Thought()` 构造函数已经自动注册到 registry，后面的 `for name, t in builtins.items(): Thought.registry[name] = t` 就是冗余的（但不会出错）。

**建议**：检查 `Thought()` 构造函数的自动注册逻辑，确认是否有重复注册的风险。

**状态**：⏳ 待检查

## 问题4：.ku 文件中的顶层语句不会自动执行

**文件**：`D:\Tools\Dao\dao\runtime.py`  
**位置**：`DaoEnv.load()` 和 `DaoEnv.run()`

**现象**：`.ku` 文件末尾可能有顶层调用语句（不在 `思` 定义内），例如：
```
读道德经()
```
`load()` 只解析 `思` 定义的函数，顶层语句被丢弃。目前天书读道德经.ku 的顶层调用 `读道德经()` 是通过 `env.run("读道德经")` 手动触发的，所以没问题。但如果期望 `.ku` 加载后自动执行顶层语句，需要额外处理。

**状态**：⏳ 按需修复
