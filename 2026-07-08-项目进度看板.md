## 1. 项目定位

道语言是一个 **AI 原生的可执行内存编程语言运行时**，北极星是：

```text
thought = code = memory   （一个"念头"同时是：可执行行为 / 可检视结构 / 可持久记忆 / 可调用工具）
```

不是通用脚本语言，也不是笔记格式。目标路径：`Python 引导 → 词法/语法 → 字节码编译 → C VM 执行 → MCP 可调用的念头运行时 → 自举编译器`。

- 早期计划 `docs/plans/2026-06-11-ku-bootstrap-phase3-compiler-ku.md` 已是约 1 个月前的旧路线，已被 Dao 线超越。

## 7. 结论与下一步

**当前坐标**：项目重心完全在 `dao/` 新线。前 5 大模块功能已到位，C VM 原生 Windows 构建（M1）与 214 项对拍测试（2026-06-15 实测全绿）已完成。当前卡点在 **M4 内存所有权模型（大件/高风险）** 与 **M5 翻转执行路径**，以及记忆留存/压实规则未定义、尚未发版。

**关键风险点**
- M4 内存模型是最大不确定项：env↔closure 成环问题未实质解决，长进程稳定性无保证。
- 根目录 20+ 临时脚本长期未清理，工作区"脏"，违背项目卫生法，且易误导后续会话"重新发明项目"。

**最小可行性第一步（建议）**
- 短期：执行 `dao_next_foundation_plan.md` 目标 C——只更新 `.gitignore` 忽略 `_*.py` 等临时文件（零风险、不动代码），先把工作区变干净。
- 中期：继续推进 M2（结构化错误）→ M3（sqlite 内置）对拍，为 M4 铺路。
