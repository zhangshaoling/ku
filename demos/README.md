# Dao C VM demos

`semantic_std_combo.kub.json` is a generated bytecode demo for the AGI semantic std modules.

It loads:

- `dao/std/semantic_core.ku`
- `dao/std/memory_env.ku`
- `dao/std/patch.ku`
- `dao/std/trace.ku`

Then it constructs a `fix_bug` thought AST, registers it into an Env, creates a Patch effect, and records it into a Trace.

`frontend_compile_demo.kub.json` is a generated bytecode demo for the bootstrap frontend.

It loads:

- `dao/std/lexer.ku`
- `dao/std/parser.ku`
- `dao/std/compiler.ku`

Then it runs `lex(source) -> parse_tokens(tokens) -> compile_ast(ast) -> run_bytecode(bytecode)` for a semantic-core-style `语义_节点` program and returns a canonical node dict. The fixture includes escaped newlines, a thought with parameters, dict mutation, a list literal, and string literals.

`frontend_bootstrap.kub.json` is the reusable C VM bootstrap image for the `.ku` frontend. It loads `lexer.ku`, `parser.ku`, and `compiler.ku` definitions without binding a fixed source program.

`golden_path.ku` is the checked-in source demo for the current project north star. It stays inside the current bootstrap compiler's stable subset and returns a concrete record for `thought = code = memory` without relying on generated JSON.

Build and run with the C VM:

```bash
gcc -o dao_core dao/dao_core.c -lm -Wall -O2
./dao_core demos/semantic_std_combo.kub.json
./dao_core --bootstrap demos/frontend_bootstrap.kub.json path/to/program.ku
./dao_core --bootstrap demos/frontend_bootstrap.kub.json demos/golden_path.ku
./dao_core --bootstrap demos/frontend_bootstrap.kub.json dao/std/semantic_core.ku path/to/program.ku
```

Regenerate the bytecode after semantic std, parser, or compiler changes:

```bash
python tools/generate_semantic_std_combo_demo.py
python tools/generate_frontend_bootstrap.py
python tools/generate_frontend_compile_demo.py
```

The semantic demo generator uses `dao/std/lexer.ku`, `dao/std/parser.ku`, and `dao/std/compiler.ku` through the current bootstrap runtime. The frontend bootstrap and demo generators use the Python parser as build harness, then emit bytecode that runs the `.ku` frontend inside the VM.
